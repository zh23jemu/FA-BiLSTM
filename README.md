# FA-BiLSTM 项目说明

## 1. 项目简介

本项目是一个基于 `FA-BiLSTM` 的测井智能识别原型，面向渤海深层潜山井段数据，主要完成以下两类任务：

- 有效储层识别
- 气层 / 含液层识别

项目同时包含一份消融实验脚本，用于对比 `LDA`、`BiLSTM` 和 `FA-BiLSTM` 在复杂干扰场景下的识别性能差异。

当前仓库以研究脚本为主，特点是：

- 使用 Excel 文件作为输入数据
- 直接在脚本中写死输入文件名、输出文件名和超参数
- 每个脚本均可单独运行
- 输出结果仍然保存为 Excel，便于后续绘图和地质解释

## 3. 功能说明

### 3.1 有效储层识别

脚本文件：

- `FA-有效储层.py`

主要功能：

- 读取全井段测井数据
- 使用 `MLR`、`AMPST`、`PHIE` 作为输入特征
- 基于 `label1` 训练 `FA-BiLSTM`
- 对整口井进行滑动窗口推演
- 输出连续储层指示曲线、平滑概率、最终预测标签和注意力权重

模型输出的核心结果包括：

- `FA_I_reservoir_Norm`
  - 归一化到 `[-1, 1]` 的连续储层指示曲线
- `FA_I_reservoir_Raw`
  - 未缩放的原始连续值
- `FA_p_reservoir_smooth`
  - 平滑后的储层概率
- `FA_pred_label1`
  - 储层识别结果
- `FA_alpha_MLR`
- `FA_alpha_AMPST`
- `FA_alpha_PHIE`

默认输出文件：

- `FA_BiLSTM_有效储层识别_全表.xlsx`


### 3.2 气液识别

脚本文件：

- `FA气液.py`

主要功能：

- 读取全井段流体评价数据
- 使用 `Sigma`、`RICX`、`RIN13`、`RATO13` 作为输入特征
- 使用 `label2` 中的已标注井段训练 `FA-BiLSTM`
- 对整口井进行连续推演
- 输出连续气层指示曲线、平滑概率、预测结果和注意力权重

标签映射关系：

- `label2 = 2` 表示气层，在训练中映射为 `0`
- `label2 = 3` 表示含液层，在训练中映射为 `1`

模型输出的核心结果包括：

- `FA_Gas_Index`
  - 连续气层指示曲线
- `FA_p_gas_smooth`
  - 平滑气层概率
- `FA_pred_label2`
  - 气液识别结果，结果重新映射回 `2` 或 `3`
- `FA_alpha_Sigma`
- `FA_alpha_RICX`
- `FA_alpha_RIN13`
- `FA_alpha_RATO13`

额外逻辑：

- 如果原始表中存在 `I_reservoir` 列，则当 `I_reservoir < 0` 时，对应深度的 `FA_Gas_Index` 会被置为 `0`

默认输出文件：

- `FA_BiLSTM_连续波动曲线_全表.xlsx`


### 3.3 消融实验

脚本文件：

- `消融实验.py`

主要功能：

- 读取带流体标签的真实样本
- 进行蒙特卡洛扩增
- 构造滑动窗口序列
- 向测试集注入“交替仪器失效”型干扰
- 对比以下模型在恶劣工况下的性能：
  - `LDA`
  - `BiLSTM`
  - `FA-BiLSTM`

输出指标包括：

- `Acc`
- `Pre`
- `Rec`
- `F1`

该脚本主要用于实验对比与论文分析，不会导出 Excel 结果，而是在终端打印评估表。


## 4. 模型思路

项目中的核心模型是 `FA-BiLSTM`，由两部分组成：

1. `BiLSTM`
   - 对井深方向上的滑动窗口序列建模
   - 提取当前深度点的上下文信息

2. `Feature Attention`
   - 对当前深度点的多项输入特征进行加权
   - 输出每个特征在当前深度点的重要性

整体流程如下：

1. 读取 Excel 数据
2. 选取目标特征列
3. 对缺失值进行前向和后向填充
4. 标准化输入特征
5. 构造长度为 `10` 的滑动窗口
6. 提取有标签样本训练模型
7. 对整口井进行连续推演
8. 将结果写回 Excel

## 6. 标签说明

### 6.1 `label1`

含义：

- 有效储层识别标签

取值：

- `0`：非有效储层
- `1`：有效储层

已知分布：

- `0` 共 `2163` 条
- `1` 共 `137` 条


### 6.2 `label2`

含义：

- 气液识别标签

取值：

- `2`：气层
- `3`：含液层

已知分布：

- `2` 共 `92` 条
- `3` 共 `45` 条

说明：

- `label2` 只在部分井段存在
- 已标注区间大致位于 `3725.996` 到 `3814.996`


### 6.3 `I_reservoir`

含义：

- 连续型储层指示值

用途：

- 作为已有解释结果或辅助约束使用
- 在 `FA气液.py` 中被用于对 `FA_Gas_Index` 做后处理

## 10. 脚本默认超参数

在两个主识别脚本中，默认超参数为：

- `WINDOW_SIZE = 10`
- `EPOCHS = 150`
- `LR = 0.01`

- 序列窗口长度为 `10`
- 深度学习模型训练轮数为 `150`


## 11. 输出结果说明

### 11.1 Excel 输出类脚本

`FA-有效储层.py` 和 `FA气液.py` 都会：

- 保留原始输入表中的字段
- 在原表基础上新增模型输出列
- 导出为新的 Excel 文件

这样便于后续：

- 绘制连续曲线
- 对比人工解释结果
- 做地质分层分析
- 做特征权重解释


### 11.2 注意力权重

输出中的 `FA_alpha_*` 列表示不同特征在当前深度点下的重要性权重，可用于辅助解释：

- 哪个测井参数在该深度段更关键
- 模型是否在某些井段自动弱化异常特征


## 12. 注意事项

1. 当前脚本中输入文件名和输出文件名是写死的
2. 代码默认从脚本所在目录读取 Excel 文件，因此运行时请保持当前工作目录为项目根目录
3. 如果要替换数据文件，需确保字段名与脚本中使用的列名一致
4. `FA气液.py` 依赖 `I_reservoir` 列做后处理，若缺失该列，脚本会打印警告
5. 项目当前没有 `requirements.txt`，如果后续需要复现或交付，建议补充依赖清单
6. 项目当前也没有统一配置文件，修改特征列、超参数或文件名时需要直接编辑脚本

## 14. 快速结论

这个项目本质上是一个面向单井测井曲线解释的时序智能识别原型：

- `FA-有效储层.py` 负责有效储层识别
- `FA气液.py` 负责气液识别
- `消融实验.py` 负责模型鲁棒性对比

输入是井深序列测井参数，输出既包括分类结果，也包括可直接用于绘图和解释的连续指示曲线。
